# 14 — 주소·랜드마크 검색 박스

지도를 손가락으로 끌어 위치를 찾던 마찰점을 검색 박스 한 줄로 해소. "어디 갈 건지" 정하는 흐름이 가장 큰 UX 병목이었고, 이 한 컴포넌트로 출발/목적지 지정이 즉시 가능해짐. URL 공유(P2.5)·즐겨찾기(P2.14) 같은 후속 기능의 진입점도 같이 열림.

## 보이는 것
- 헤더 바로 아래 새 검색 박스 + ✕ 클리어 버튼 + "검색" 라벨
- "명동성당" 검색어에 Nominatim이 돌려준 3건 드롭다운 (다크 카드, 한국 주소 그대로)
- 칩 행은 그대로 (전체/일반/재활용/내 위치/직선/출발 탭/🏁 목적지/보행 속도/타일 테마)
- 통계 바: `📍 59 / 전체 59개`
- 다크 타일 + 다중 휴지통 마커 (혼합 보라 🗑️♻️ + 단일 일반 🗑️)

## 무엇이 끝났나
- `src/lib/search.ts` — Nominatim 호출 분리 (순수 함수, 타입 명시)
- `src/app/api/search/route.ts` — 내부 프록시 (User-Agent 강제, `cache: 'no-store'`)
- `src/components/SearchBox.tsx` — 300ms debounce, 최대 5건 드롭다운, 로딩/빈 결과/에러 분기, 외부 클릭 닫기, 44px 터치 타겟
- 출발/목적지 탭 모드 연동 — 결과 클릭 시 모드에 맞게 좌표 set + 지도 panTo
- `bun run build` 통과 (Turbopack 836ms, TypeScript strict)
