# 04 — 필터 + 위치 결합 동작

03이 P2.1 기능 단독을 보여줬다면, 04는 **두 시스템(필터·위치)이 함께 동작함**을 한 화면으로 증명. 사용자 좌표는 정확히 JG-0009 (중림로10 가로변, 일반-only) 위치. 재활용 필터를 켜면 그 통이 후보에서 빠지므로 nearest가 다른 통으로 점프해야 한다 — 그 동작을 시각화.

## 보이는 것
- **재활용 칩**: 초록 active (`#10b981`)
- **위치 끄기 칩**: 파랑 active (`#0ea5e9`)
- **통계**: `📍 56 / 전체 59개 · 가까운 통 88m (종근당 충정로역 …)` (sky-300 라벨)
- **사용자 펄스 점**: 중림로10 좌표 (지도 중앙 근방)
- **노란 강조 ring**: 종근당 충정로역(JG-0012) 위치 — 88m 떨어진 곳. 일반-only인 JG-0009가 빠지면서 다음 후보로 점프
- 보이는 마커는 모두 보라(혼합 56). 일반-only 3개는 필터에 의해 제외

## 검증한 것
이번 세션에서 5개 좌표 + 필터 결합 케이스로 검증:
- Haversine 거리 = UI 라벨 일치 (서울시청 103m, 명동성당 121m, DDP 52m, 서울역광장 430m, 강남역 6.3km)
- formatDistance m/km 자동 전환 (1km 임계)
- 필터 변경 시 nearest 재계산 정상 (전체→0m, 재활용→88m, 일반→0m)
- 강조 ring + 사용자 마커 모두 표시
- 토글 (위치 끄기 → 다시 켜기) 정상
